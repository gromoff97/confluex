# AGENTS

- IDENTIFY THE OWNING TEST LAYER BEFORE EDITING.
- IF AREA IS UNCLEAR, READ `docs/AGENTS.md` FOR AREA MAP.
- READ THE RELEVANT `docs/FR-<AREA>.md` BEFORE TEST EXPECTATION REVIEW OR TEST EXPECTATION CHANGES.
- IF IMPLEMENTATION BEHAVIOR IS IN SCOPE, READ `lib/AGENTS.md`.
- ADD OR UPDATE TESTS FOR CHANGED BEHAVIOR AND CHANGED REQUIREMENTS.
- KEEP TESTS ALIGNED WITH CANONICAL REQUIREMENTS.
- DO NOT INVENT BEHAVIOR THAT IS NOT DEFINED IN `docs/FR-<AREA>.md`.
- IF TEST REVIEW OR TEST WORK TOUCHES BEHAVIOR OR REQUIREMENTS, APPLY ROOT `AGENTS.md` REVIEW CHECKLIST.
- RUN THE SMALLEST RELEVANT TEST SET FIRST.
